- Mejora: de N requests HTTP a 1 solo request (N=prospectos)
- Compatibilidad: mantiene endpoint single insert para compatibilidad
- Endpoint DELETE /api/prospectos sin ?id= borra todos (para replaceAll)

AUTENTICACIÓN MULTI-USUARIO (12/07/2026)
============================================================

- Migración de autenticación por API Key a sistema multi-usuario con usuario/contraseña
- Nuevas tablas BD: `usuarios` (id, nombre único, email, password_hash SHA256, rol, activo) y `usuarios_clientes` (N:M usuarios ↔ clientes)

- Header `X-Cliente-Id` reemplaza `X-API-Key` en todas las API Routes
- Helper `lib/auth-utils.ts` con `getClienteId(req)` para extraer cliente_id del header
- `services.ts` actualizado para enviar `X-Cliente-Id` automáticamente
- `store.tsx`: estado global incluye `user` (Usuario | null), acciones `SET_USER` / `LOGOUT`
- Session restore: AppProvider lee `mercurio_user` de localStorage, valida con GET /api/cliente?cliente_id=...
- LoginForm: campos usuario + password + link a registro
- Register page: usuario, email opcional, password (mín 6), API Key del cliente
- Logout: limpia `mercurio_user` + dispatch LOGOUT + redirect a /
- 6 API Routes migradas a `X-Cliente-Id`: cliente, plantillas, prospectos, mensajes, send-form-data, variables
- Trigger `validate_usuario_cliente` en BD para integridad referencial
