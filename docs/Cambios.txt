=== MERCURIO SEND — REGISTRO DE CAMBIOS ===

Fecha: 9/07/2026

============================================================
MIGRACIÓN COMPLETA A NEXT.JS 14 + API ROUTES
============================================================

- Prototipo vanilla HTML/CSS/JS movido a legacy/
- App migrada a Next.js 14 con TypeScript
- API Routes implementadas para toda la lógica:
  * GET /api/cliente — validación API Key con SHA256
  * GET /api/variables — variables de configuración
  * GET|POST|PUT|DELETE /api/plantillas — CRUD plantillas
  * GET|POST|PUT|DELETE /api/prospectos — CRUD prospectos
  * GET /api/mensajes — historial de mensajes
  * POST /api/send-message — envío de templates
  * POST /api/send-media — envío de imágenes/video
  * GET|POST /api/webhook — webhook Meta
  * POST /api/outbound — registro manual de salida
  * POST /api/chatwoot/webhook — webhook Chatwoot
  * GET|POST /api/send-form-data — persistencia de formulario

============================================================
ARQUITECTURA DE COMUNICACIÓN
============================================================

Cambio fundamental: toda comunicación ahora pasa por API Routes.
El navegador NO habla directo a Supabase SDK.

Anterior:
  Navegador → Supabase SDK (directo a PostgreSQL via REST)

Actual:
  Navegador → API Route (Vercel/localhost) → Supabase
